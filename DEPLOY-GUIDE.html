🚀 Hugo 博客部署指南

Hugo + PaperMod + Cloudflare Pages — 全程免费，全球加速

1. 安装 Hugo
2. 克隆主题
3. 推送 GitHub
4. 连接 Cloudflare
5. 完成
安装 Hugo 第 1 步 / 共 4 步

Hugo 是一个用 Go 编写的静态网站生成器，需要先安装到你的电脑上。

Windows 用户（推荐用 Scoop）：

# 1. 先安装 Scoop（如果没有）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
irm get.scoop.sh | iex

# 2. 安装 Hugo Extended 版本
scoop install hugo-extended

或者直接下载安装包：

前往 https://github.com/gohugoio/hugo/releases，下载最新版本的 hugo_extended_xxx_windows-amd64.zip，解压后将 hugo.exe 添加到系统 PATH 环境变量中。

验证安装：

hugo version
# 应该显示类似：hugo v0.147.2+extended ...

同时确保已安装 Git：

git --version
# 如果没有，前往 https://git-scm.com 下载安装
克隆 PaperMod 主题 第 2 步 / 共 4 步

博客文件已经在 my-blog 文件夹中准备好了，现在需要克隆 PaperMod 主题。

1
进入博客目录
cd my-blog
2
初始化 Git 并克隆主题
git init
git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
这会把 PaperMod 主题作为 Git 子模块添加进来，Cloudflare Pages 部署时会自动拉取。
3
本地预览博客
hugo server -D
# 打开浏览器访问 http://localhost:1313

博客文件结构：

my-blog/
  ├── hugo.toml    ← 主配置文件
  ├── wrangler.toml ← Cloudflare 构建配置
  ├── .gitignore
  ├── content/
  │   ├── posts/ ← 在这里写文章
  │   │   └── hello-world.md
  │   ├── about/
  │   │   └── index.md
  │   └── search.md
  ├── themes/
  │   └── PaperMod/ ← 主题目录
  ├── static/ ← 静态文件
  └── .github/workflows/
       └── deploy.yml
创建 GitHub 仓库并推送 第 3 步 / 共 4 步
1
在 GitHub 创建新仓库

前往 github.com/new，创建一个名为 my-blog 的仓库（选 Public 或 Private 都可以），不要勾选 "Initialize this repository with a README"。

2
推送本地代码到 GitHub
# 在 my-blog 目录下执行
git add .
git commit -m "feat: 初始化博客"
git branch -M main
git remote add origin https://github.com/你的用户名/my-blog.git
git push -u origin main
记得把 你的用户名 替换成你的真实 GitHub 用户名！
连接 Cloudflare Pages 第 4 步 / 共 4 步
1
登录 Cloudflare Dashboard

前往 dash.cloudflare.com，在左侧菜单找到 Workers & Pages → Create application → Pages → Connect to Git

2
连接 GitHub 仓库

授权 Cloudflare 访问你的 GitHub 账号，然后选择刚才创建的 my-blog 仓库。

3
配置构建设置

填写以下构建参数：

Framework presetHugo Build commandhugo --minify Build output dirpublic Root directory（留空）
4
添加环境变量

在 Environment variables 中添加：

HUGO_VERSION0.147.2
5
点击 Save and Deploy

Cloudflare 会自动构建并部署你的博客，通常 1~2 分钟内完成。部署成功后会获得一个 xxx.pages.dev 的访问地址。

之后每次 git push 到 main 分支，Cloudflare Pages 会自动重新部署！
🎉

恭喜！博客搭建完成

你的 Hugo 博客已经部署到 Cloudflare Pages，全球 CDN 加速，完全免费！

接下来可以做的事
✓
写新文章

在 content/posts/ 目录下创建 Markdown 文件，推送到 GitHub 后自动部署。

hugo new posts/我的新文章.md
✓
更新博客信息

编辑 hugo.toml 文件，修改博客标题、作者名、社交链接等。别忘了修改 baseURL 为你实际的 pages.dev 地址。

✓
绑定自定义域名

如果你有自己的域名，在 Cloudflare Pages 项目设置中添加 Custom domains，然后在 DNS 中添加 CNAME 记录指向 xxx.pages.dev 即可。

✓
添加评论系统（可选）

PaperMod 支持 Disqus、utterances（GitHub Issues）等评论系统，在 hugo.toml 中配置即可。
